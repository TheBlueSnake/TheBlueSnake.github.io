本文主要介绍TheBlueSnake 撰写博客或者使用 markdown 时的规范。有参考洛谷题目 markdown 排版要求。可能会随着时间和吸管的改变更新。对于不符合规范的文章在退役后可能会随缘规范化。

常用规范说明

  1. 不做特殊处理：不需要任何处理，当做普通的字符和句子一样。
  2. 按需求空格：如果左（右）边是普通字符，则应该空一格。如果是汉字逗号 ， 那么可以不空格。

粗体、斜体 、划线 的使用规范

  1. 粗体 使用 **something** 语法来实现，在使用时，不做特殊处理。
  2. 斜体 使用 *something* 语法来实现，在使用时，由于字体向右倾斜，右侧按需求空格。
  3. 划线 使用 ~~something~~ 语法来实现，在使用时，左右两侧按需求空格。

代码块的使用规范

  1. 小代码块使用 `````` 语法来实现，在使用时，左右两侧按需空格。
  2. 大代码块将两者拆开来使用，最后必须注明语言或者 text 文本类型，由于是大型块结构，注意上下两行应该空行。

$\LaTeX$ 使用规范

  1. 具体 $\LaTeX$ 语法遵守其官方语法规范，内部可以空格，也可以不空格，但是同一 $\LaTeX$ 块中间不允许混用。
  2. $\LaTeX$ 使用 $$\LaTeX$$ 语法来实现，在使用时，左右两侧按需求空格。
