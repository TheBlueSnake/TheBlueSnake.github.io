题目描述

2048年，第三十届 CSP 认证的考场上，座位选手的小明打开了第一题。这个题的样例有 $n$ 组数据，数据从 $1\sim n$ 编号， $i$ 号数据的规模是 $a_i$。

小明对该题设计出了一个暴力程序，对于一组规模为 $u$ 的数据，该程序的运行时间为 $u^2$。然而这个程序运行完一组规模为 $u$ 的数据之后，它将在任何一组规模小于 $u$ 的数据上运行错误。样例中的 $a_i$ 不一定递增，但小明又想在不修改程序的情况下正确运行样例，于是小明决定使用一种非常原始的解决方案：将所有数据划分成若干个数据段，段内数据编号连续，接着将同一段内的数据合并成新数据，其规模等于段内原数据的规模之和，小明将让新数据的规模能够递增。

也就是说，小明需要找到一些分界点$1\leq k_1<k_2<\cdots<k_p<n$，使得

注意 $p$ 可以为$0$，且此时$k_0=0$，也就是说小明可以将所有数据合并在一起运行。

小明希望他的程序在正确运行样例情况下，运行时间也能尽量小，也就是最小化

小明觉得这个问题非常有趣，并向你请教：给定 $n$ 和 $a_i$，请你求出最优划分方案下，小明的程序的最小运行时间。

输入格式

由于本题的数据范围较大，部分测试点的 $a_i$ 将在程序内生成。

第一行两个整数 $n$，$type$ 。$n$ 的意义见题目描述，$type$ 表示输入方式。

  1. 若 $type=0$，则该测试点的 $a_i$ 直接给出。输入文件接下来：第二行 $n$ 个以空格分割的整数 $a_i$ ，表示每组数据的数据规模。
  2. 若 $type=1$，则该测试点的 $a_i$ 将特殊生成，生方式见后文。输入文件接下来：第二行六个以空格分隔的整数 $x$，$y$，$z$，$b_1$，$b_2$，$m$。接下来 $m$ 行中，第 $i(1\leq i\leq m)$ 行包含三个以空格分隔的正整数 $p_i$，$l_i$，$r_i$。

对于 $type=1$的 $23\sim25$ 号测试点，$a_i$ 的生成方式如下：

给定整数 $x$，$y$，$z$，$b_1$，$b_2$，$m$，以及 $m$ 个三元组 $(p_1,l_i,r_i)$。

保证 $n\geq 2$。若 $n>2$，则 $\forall 3\leq i\leq n,b_i=(x\times b_{i-1}+y\times b_{i-2}+z)\mod 2^{30}$。

思路与解法

首先考虑前 $1\sim 9$ 个点的解法。我们用 $f_{i,j}$ 表示前 $i$ 个测试点， 上一个测试点选在 $j$ 时最小的花费。考虑如下dp转移方程：

这样很简单做 $O(n^3)$ 就可以获得 $36$ 分了。随后考虑优化这个转移方程，我们经过打表可以发现 $f_{i,j}$ 在 $i$ 固定的时候，随 $j$ 的变小逐渐变大。这里给出该结论的一种证明方法：

基本不等式：如果 $\forall a,b\in \mathbb{R^+}$，那么有 $(a+b)^2=a^2+b^2+2ab\geq a^2+b^2$。

那么我们考虑存在测试点 $i>j>k$，以及一个正实数 $p$。满足 $sum_{i}-sum_{j}\geq sum_{j} -sum_{k}$ 且 $sum_{i}-sum_{j}\geq sum_{j}-sum_{j-p}$。那么 $f_{i,j}=\min\left(f_{j,k},f_{j,k-p}\right)+\left(sum_{i}-sum_{j}\right)^2$，$f_{j,k}\leq (sum_{j}-sum_{k})^2+(sum_{k}-sum_{k-p})^2+f_{k-p,pos}$，而 $f_{j,k-p}=(sum_{j}-sum_{k-p})^2+f_{k-p,pos}$，因此可以得出 $f_{j,k}\leq f_{j,k-p}$ 的结论，证毕。

至此，我们获得了一个答案的单调性。对于状态 $f_{i,j}$，我们可以优化为 $f_i$ 表示前 $i$ 个测试点，上一个测试点最大的时候，即时间花费正好最小的时候的花费是多少。那如果我们使用 $pre_i$ 表示前 $i$ 个测试点最接近 $i$ 的转移位置的话，我们可以获得如下的转移方程：

这样做的复杂度是 $O(n^2)$，可以通过 $1\sim 16$ 的测试点，对于接下来的测试点很明显我们需要优化成 $O(n)$ 的dp，考虑单调队列。我们这时候发现计算转移的重点从 $f_i$ 转变成了 $pre_i$。因为我们可以很轻松地通过 $pre_i$ 计算出答案：

我们发现剩下的数据点要求 $O(n)$ 的复杂度，所以考虑使用单调队列优化。如果要使用单调队列的话我们还需要 $pre_i$ 的单调性，可以通过打表发现 $pre_i$ 是单调不降的，这里同样给出一种证明方法：

我们将 $pre_i$ 的转移方程变换一下，条件由 $sum_{j}-sum_{pre_j}\leq sum_{i} - sum_{j}$ 变为 $sum_{i}\geq 2\times sum_{j} - sum_{pre_j}$。那么如果存在 $1\leq i<j\leq n$，那么我们一定存在 $sum_{j}\geq 2\times sum_{pre_{j}}-sum_{pre_{pre_{j}}}$。那么由前缀和的定义我们可以得到 $sum_{i}\geq sum_{j}\geq 2\times sum_{pre_{j}}-sum_{pre_{pre_{j}}}$，所以当 $pre_{i}=pre_{j}$ 的时候依旧满足条件，所以 $pre_{i}\geq pre_{j}$。

所以我们使用单调队列优化维护 $pre_{i}$，可以做到 $O(1)$ 转移，总体复杂度 $O(n)$，可以通过全部数据。

注意由于我们维护的是最小值，所以单调队列队首的滑动窗口在滑动的时候要先判断队首后的位置是否可行然后再进行滑动。因为太弱所以在这里卡了半个小时。

以下是使用了__int128的程序：

#include<bits/stdc++.h>
using namespace std;
const int MAXN = 40000100;
const int MOD = 1 << 30;
typedef long long ll;

int n, type;
int arr[MAXN], b[MAXN];
int f[MAXN];
ll sum[MAXN];
int pre[MAXN], d[MAXN];
int hd, tl, que[MAXN];

void input(int flag){
    if(flag){
        int x, y, z, m;
        scanf("%d %d %d %d %d %d", &x, &y, &z, &b[1] ,&b[2], &m);
        for(int i = 3; i <= n; i++){
            b[i] = (1ll * x * b[i - 1] + 1ll * y * b[i - 2] + z) % MOD;
        }
        int prev = 0, xi;
        int l, r;
        for(int i = 1; i <= m; i++){
            scanf("%d %d %d", &xi, &l, &r);
            for(int j = prev + 1; j <= xi; j++){
                arr[j] = (b[j] % (r - l + 1)) + l;
            }
            prev = xi;
        }
    }else{
        for(int i = 1; i <= n; i++){
            scanf("%d", &arr[i]);
        }
    }
}

int main(){
    scanf("%d %d", &n, &type);
    input(type);
    sum[0] = 0;
    for(int i = 1; i <= n; i++){
        sum[i] = sum[i - 1] + arr[i];
    }
    hd = 1; tl = 0;
    que[++tl] = 0;
    for(int i = 1; i <= n; i++){
        while(hd < tl && 2ll * sum[que[hd + 1]] - sum[pre[que[hd + 1]]] <= sum[i]) hd++;
        pre[i] = que[hd];
        while(hd <= tl && 2ll * sum[i] - sum[pre[i]] <= 2ll * sum[que[tl]] - sum[pre[que[tl]]]) tl--;
        que[++tl] = i;
    }
    int now = n; __int128 ans = 0, tmps = 1;
    while(now){
        tmps = sum[now] - sum[pre[now]];
        tmps *= sum[now] - sum[pre[now]];
        ans += tmps;
        now = pre[now];
    }
    int res[50], head = 0;
    while(ans){
        res[++head] = ans % 10;
        ans = ans / 10;
    }
    while(head){
        printf("%d", res[head--]);
    }
    printf("\n");
    return 0;
}
