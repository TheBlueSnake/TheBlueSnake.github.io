题面

题目描述

给一个由小写字母组成的字符串，我们可以用一种简单的方法来压缩其中的重复信息。压缩后的字符串除了小写字母外还可以（但不必）包含大写字母R与M，其中M标记重复串的开始，R重复从上一个M（如果当前位置左边没有M，则从串的开始算起）开始的解压结果（称为缓冲串）。


bcdcdcdcd可以压缩为bMcdRR，下面是解压缩的过程：

已解压的部分 解压结果 缓冲串
b b b
bM b 无
bMc bc c
bMcd bcd cd
bMcdR bcdcd cdcd
bMcdRR bcdcdcdcd cdcdcdcd

输入格式

输入仅一行，包含待压缩字符串，仅包含小写字母，长度为$n$。

输出格式

输出仅一行，即压缩后字符串的最短长度。

输入样例1

aaaaaaa

输出样例1

5

输入样例2

bcdcdcdcdxcdcdcdcd

输出样例2

12

说明/提示

在第一个例子中，解为aaaRa。在第二个例子中，解为bMcdRRxMcdRR。

数据范围

$50\%$的数据满足：$1\leq n\leq 20$

$100\%$的数据满足：$1\leq n\leq 50$

题解 & 分析

很有趣的一道题，看起来和P4302 [SCOI2003]字符串折叠很像，但是仔细自己做一做就会发现完全不一样。

首先依然是同样考虑区间dp，最开始想的肯定是用f[i][j]表示从i至j的最小值，如果你用和P4302类似的转移方程的话， 你会发现在样例2就无法通过。原因是这道题的R重复上一个M，所以说如果出现比如MabcdMefRR的串，它的解压结果是abcdefefefef，和之前想的不一样。

我们会发现该串看起来是将一部分重复$2^x$次，随后将它们拼接起来，所以考虑先计算$2^x$的拼接结果，然后用$f_{i,j}=\min_{i\leq k<j}\left\{f_{i,k}+f_{k+1,j}\right\}$的转移方程将它们拼接起来。这种做法我们通过了两个样例，交一发会发现是$60$分。原因是会出现同开头的嵌套时没有关系的，比如ababababcabababc可以被压缩为MabRRcR即可。

所以接下来考虑正解，状态f[i][j][0]表示从i至j，保证i前有一个M，且保证i至j之间不存在M的情况下，可以压缩成的最小长度。状态f[i][j][1]表示从i至j，保证i前有一个M，在i至j之间存在M的情况下，可以压缩成的最小长度。

正确性证明

转移方程$(1)$，枚举其切开的位置k，对于i至k的字符，由于位置i前面有一个M，采用最佳压缩方$f_{i,k,0}$，但是由于从i至j之间不存在M，我们考虑将其原封不动地复制过来，花费是$j-(k+1)+1=j-k$。

转移方程$(2)$，枚举中间M的位置k，那么开头从i至k的字符，花费最小就是$\min\left(f_{i,k,0},f_{i,k,1}\right)$。然后我们在位置k后放置一个M，从k + 1至j的字符的最小花费就变成了$\min(f_{k+1,j,0},f_{k+1,j,1})$。于是全部的最小花费就是$\min\left(f_{i,k,0},f_{i,k,1}\right)+\min\left(f_{k+1,j,0},f_{k+1,j,1}\right)+1$，最后一个$1$就是中间加上的M的长度。

转移方程$(3)$，即当前位置可以被切开，从i至$\frac{i+j}2$，从$\frac{i+j}2+1$至j两部分完全相同，此时要求中间不能存在干扰的M，否则最后加入的R会干扰最终的答案，使解压结果不正确，花费只会在前半段的基础上多一个R因此直接加一即可。

AC代码

#include<bits/stdc++.h>
using namespace std;
const int MAXN = 64;

int n;
char str[MAXN];
int f[MAXN][MAXN][2];

bool chk(int i, int j){
    if((j - i + 1) % 2 != 0) return false;
    int mid = (i + j) >> 1;
    for(int p = i; p <= mid; p++){
        if(str[p] != str[p + mid - i + 1]){
            return false;
        }
    }
    return true;
}

int main(){
    scanf("%s", str + 1);
    n = strlen(str + 1);
    memset(f, 0x3f, sizeof(f));
    for(int i = 1; i <= n; i++)
        for(int j = i; j <= n; j++)
            f[i][j][0] = f[i][j][1] = j - i + 1;
    for(int l = 2; l <= n; l++){
        for(int i = 1; i + l - 1 <= n; i++){
            int j = i + l - 1;
            if(chk(i, j)){
                f[i][j][0] = min(f[i][(i + j) / 2][0] + 1, f[i][j][0]);
            }
            for(int k = i; k <= j - 1; k++){
                f[i][j][0] = min(f[i][j][0], f[i][k][0] + j - (k + 1) + 1);
                f[i][j][1] = min(
                             f[i][j][1],
                             min(f[i][k][0], f[i][k][1]) + min(f[k + 1][j][0], f[k + 1][j][1]) + 1
                             );
            }
        }
    }
    printf("%d\n", min(f[1][n][0], f[1][n][1]));
    return 0;
}
